# AI coding agents — fundamentals

A practical, vendor-neutral guide in 12 chapters on how to start working seriously with AI coding agents (Claude Code, Codex, Copilot, Cursor, etc.): agents, skills, slash commands, hooks, MCP, permissions, memory, and the anti-patterns of the first months.

> The model is the engine. But before you build a *harness* around it, you need to understand the parts: what an agent really is, what skills and commands are for, how context flows, and where the loop closes.

## About the guide

Written for technical teams that are **just starting** to operate AI coding agents and want a solid mental model before they hit scaling pain. It does not assume prior experience with agents — only with software engineering.

When your team outgrows this guide and starts feeling the pain of running agents at scale, the natural next step is the companion guide:

👉 **[Harness Engineering — building software at scale with agents](https://github.com/jmlopezdona/ai-coding-agents-harness)**

## What this guide is *not*

- Not a vendor manual. Examples lean on Claude Code because it's the most documented today, but the concepts apply to Codex, Cursor, Copilot and others.
- Not a prompt cookbook. The goal is the mental model, not magic incantations.
- Not the harness guide. If your team already operates agents and is hitting scaling pain, jump to the harness guide directly.
