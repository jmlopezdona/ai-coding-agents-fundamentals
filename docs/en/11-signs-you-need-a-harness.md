# 11. Signs you need a harness

This guide is enough to get your team productive with agents. It is *not* enough to keep you productive once you scale. At some point you'll feel friction that no amount of prompt-tweaking fixes — that's the moment to graduate to harness engineering.

## What you'll learn

- Concrete symptoms that mean "you've outgrown this guide."
- Why the fix is structural, not prompt-level.
- Where to go next.

## Outline

1. **Symptom: the agent repeats the same mistakes across developers.** No shared correction loop.
2. **Symptom: every developer has their own setup.** No shared harness.
3. **Symptom: agent PRs need more review than human PRs.** Verification has been pushed onto the human instead of the harness.
4. **Symptom: the same bug class shows up over and over.** No sensors catching it before review.
5. **Symptom: onboarding a new dev to "the agent way" takes a week.** No system of record.
6. **Symptom: you're writing longer and longer prompts.** You're trying to compensate for missing guides and sensors.
7. Why prompt-engineering won't fix any of these: the problem is the absence of a system.

## What's next

When you recognize these symptoms, jump to the companion guide:

👉 **[Harness Engineering — building software at scale with agents](https://jmlopezdona.github.io/ai-coding-agents-harness/)**

It picks up exactly where this one stops: how to build the guides, sensors, loops, sandboxes and structured context that turn an LLM into an agent a team can rely on.

## Key takeaway

If you're tweaking prompts to fix structural problems, you've outgrown this guide. The next step isn't a better prompt — it's a harness.
