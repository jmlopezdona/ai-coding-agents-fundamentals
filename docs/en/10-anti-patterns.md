# 10. Anti-patterns of the first months

The same mistakes show up in almost every team's first months with agents. None of them are catastrophic on their own; together they make the difference between a team that gets value and one that quietly gives up.

## What you'll learn

- The most common early anti-patterns and how to recognize them.
- Why each one is tempting and what it costs.
- The minimum-effort fix for each.

## Outline

1. **The mega-prompt** — solving everything in one giant message instead of iterating.
2. **The 2000-line `AGENTS.md`** — encyclopedia instead of instructions.
3. **"Please be careful"** — asking the agent to remember instead of using a hook.
4. **Disabling permissions to go faster** — speed today, pain tomorrow.
5. **Reviewing agent PRs less than human PRs** — exactly backwards.
6. **No team conventions** — every developer with their own setup, no shared commands or skills.
7. **Treating memory as a notebook** — see chapter 9.
8. **Trusting tool output blindly** — including possible prompt injection in fetched content.

## Key takeaway

None of these are clever traps. They're all "the obvious lazy choice." Knowing they're anti-patterns is most of the cure.
